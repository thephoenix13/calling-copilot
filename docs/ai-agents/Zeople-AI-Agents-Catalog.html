01

Pipeline Sessions

Agentic 7-step hiring drive — Steps 2 through 6 each invoke AI agents.

10 agents
JD Enhancer
POST /enhance-jd
TriggerStep 2 → recruiter clicks "Generate Assets"
Inputsjob, clientNotes, companyScript, fields
OutputsformattedJD · recruiterBrief · clarificationQuestions · reachoutMaterial · sourcingKeywords · marketIntelligence
ModelClaude Sonnet 4.6
Fires1× per session (shared by all candidates)
Resume Quality Evaluator
POST /sessions/:id/candidates/:cid/evaluate
TriggerStep 3 → auto-fires when candidate is added
Inputsresume text, job title, required & preferred skills
Outputsoverall · contentQuality · aiWritingDetection · experienceLevel · verdict · strengths
ModelClaude Haiku 4.5
Fires1× per candidate added to the session
AI Content Detector
POST /ai/check-ai-content
TriggerStep 3 → recruiter clicks "Run AI Check" (also exposed standalone in Candidate Database)
Inputsresume text (100–3000 chars)
Outputsverdict (human / likely_human / mixed / likely_ai / ai_generated) · confidence · indicators · summary
ModelClaude Haiku 4.5
FiresOn demand · 0–1 per candidate
VI Question Generator
POST /vi/interviews/:id/generate
TriggerStep 5a → recruiter creates a video interview template
Inputsjob description, desired question count
Outputsarray of { question_text, question_type (technical/behavioral/situational), estimated_time_minutes }
ModelClaude Sonnet 4.6
Fires1× per interview template (shared across all VI candidates)
VI Evaluator
POST /vi/candidates/:cid/evaluate
TriggerStep 5a → recruiter clicks "Evaluate" on a completed video interview
Inputsjob description, transcribed candidate responses (Deepgram)
Outputsoverall_score · hiring_recommendation · evaluation_summary · strengths · weaknesses · 5-competency scores · per-question evaluations
ModelClaude Sonnet 4.6
Fires1× per candidate per interview
MCQ Question Generator
POST /assessments/:id/ai-generate
TriggerStep 5b → recruiter clicks "Generate Questions"
Inputsjob context, recruiter brief & phone screening Qs (from Step 2), topic & difficulty preference
Outputsup to 10 MCQs each with { question_text, 4 options, correct_option, explanation, topic, difficulty }
ModelClaude Sonnet 4.6
Fires1× per MCQ assessment (shared by all invitees)
MCQ Evaluator
(internal) /assessments/take/:token/submit
TriggerStep 5b → fires automatically after candidate submits
Inputsassessment title, score breakdown, per-question correctness
Outputssummary · strengths · gaps · topic_scores · difficulty_breakdown · recommendation · question_analysis
ModelClaude Haiku 4.5
Fires1× per submission · async (does not block response)
Coding Question Generator
POST /coding-assessments/:id/ai-generate
TriggerStep 5c → recruiter clicks "Generate Questions"
Inputslanguage, count, difficulty, job context
Outputsarray of { title, problem_statement, starter_code, language, question_type (write/fix/complete), difficulty }
ModelClaude Sonnet 4.6
Fires1× per coding assessment (auto-balances 40/30/30 across question types)
Coding Evaluator
(internal) /coding-assessments/take/:token/submit
TriggerStep 5c → fires automatically after candidate submits
Inputscode solutions per question, language, difficulty, type metadata
Outputsoverall_score · summary · recommendation · strengths · gaps · per-question { correctness, quality, feedback }
ModelClaude Haiku 4.5
Fires1× per submission · async
Call QA + Candidate Report
POST /ai/generate-reports
TriggerStep 6 → fires when a recruiter call ends (and from Calling CoPilot)
Inputstranscript (speaker/text), role, candidateName, recruiterName, JD excerpt, resume excerpt
OutputsqaReport (8-dim scorecard, red flags, weak/better nudges) + candidateReport (verdict, scores, recommendation)
ModelClaude Opus 4.6
Fires1× per completed recruiter call · powers the Recruiter QA module
02

Calling CoPilot

Real-time AI coaching during live recruiter calls + the auto-demo simulation.

3 agents
Live Follow-Up Suggester
POST /ai/suggest
TriggerCalling CoPilot → debounced ~1s after each candidate response
Inputslast ~6 candidate utterances, current question context
Outputs2–3 sharp follow-up questions
ModelClaude Haiku 4.5
FiresOnce per new candidate transcript line · low-latency requirement
Candidate Simulation Reply
POST /ai/sim-reply
TriggerDemo Vid / Auto Demo flow — generates a candidate response after each recruiter question
Inputsrecruiter message, conversation history (last 10 turns), role, candidate name, JD, resume
Outputs{ candidateReply: 2–4 sentences in candidate persona }
ModelClaude Haiku 4.5
Fires1× per recruiter question in a simulated call (~8–10 per demo)
Interview Question Generator
POST /ai/generate-questions
TriggerCalling CoPilot → recruiter clicks "Generate Guide" in the Interview Panel
InputsJD (file or text) + resume (optional)
Outputs4–6 sections (Background · Technical · Problem-Solving · Role-Specific · Culture · Closing) with 3–5 questions each
ModelClaude Opus 4.6
Fires1× per interview guide build · pre-call preparation

Job Management

Hiring-manager intake and per-job JD asset generation.

2 agents
HM Clarification Question Generator
POST /jobs/:id/qualification-questions
TriggerJob detail → recruiter clicks "Generate Intake Questions" (or regenerates a single Q)
Inputsjob title, client, required skills, description · (optional) regenerateIndex + existingQuestions

TriggerRecruiter saves HM Q&A and marks the job qualified
Inputsjob details + HM Q&A answers
OutputsRefreshed JD assets (formattedJD · recruiterBrief · sourcingKeywords · reachoutMaterial · clarificationQuestions)
ModelClaude Sonnet 4.6
Fires1× per job qualification · async background refresh

05

Post-Offer Follow-Up (POFU)

Auto-generated email outreach across the offer-to-Day-1 lifecycle.

1 agent
POFU Email Composer
server/utils/pofu-engine.js → generateEmail()
TriggerPOFU engine selects a trigger (weekly check-in / silence follow-up / DOJ reminder / BGV request / risk escalation / no-response nudge) and asks Claude for a message
Inputscandidate name, role, company, DOJ, current state, risk level, trigger reason
Outputs{ subject, body } — plain-text email signed "The Zeople Team"
ModelClaude Haiku 4.5
FiresMultiple per candidate over the offer-to-joining window (~3–6 emails typical)
Notes. The Pipeline Sessions count (10 agents) includes the Call QA + Candidate Report agent that also runs from Calling CoPilot — it is listed once under its primary triggering surface. The AI Content Detector appears in two sections (Pipeline Step 3 and Candidate Database) since the same endpoint is called from both surfaces; counted once for the unique-endpoint total. Async evaluators (MCQ, Coding, POFU) fire after the user-facing response is returned, so user-perceived latency is unaffected by their cost.
